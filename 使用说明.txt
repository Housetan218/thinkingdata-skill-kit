ThinkingData Skill 使用说明

GitHub 长期使用推荐方式：

1. 直接把仓库 clone 到 skill 目录
2. 以后直接 git pull 更新

示例：

- ~/.agents/skills/
  - git clone <repo-url> ~/.agents/skills/thinkingdata-analysis-entry
- ~/.codex/skills/
  - git clone <repo-url> ~/.codex/skills/thinkingdata-analysis-entry

如果只是下载压缩包，再使用下面的兼容安装方式：

1. 打开“终端”
2. 输入 bash 后留一个空格
3. 把 install.sh 文件直接拖进终端窗口
4. 按回车执行
5. 重新打开 Codex
6. 在对话里输入：

请使用 thinkingdata-analysis-entry skill。先只检查登录态和项目列表，不要打开 SQL；等我给你项目、日期、指标后再执行查询。

说明：
- 安装目标目录为 ~/.agents/skills/thinkingdata-analysis-entry/
- 如果你的 Codex 环境使用 ~/.codex/skills/，请执行：
  - bash install.sh --codex-home
- 安装时会把整个 skill 目录一起复制，包括：
  - SKILL.md
  - scripts/run_sql_via_ws.js
  - scripts/scaffold_project_kit.sh
  - scripts/record_timing_log.sh
  - runbooks/
  - references/
- 如果 Codex 已经开着，安装后请新开会话再使用
- install.command 可能会被 macOS 拦截，install.sh 更稳
- Browser Use CLI 只负责浏览器侧辅助，不替代 SQL 主链路
- Playwright CLI 是默认浏览器执行器；Browser Use CLI 只是 fallback
- 不要只说“开始分析”，最好直接补齐：
  - 项目或产品
  - 日期或日期范围
  - 指标
  - 输出格式
